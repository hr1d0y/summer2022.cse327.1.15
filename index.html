Introduction


Purpose:

The purpose of this srs is to describe a system to take online exams. The main agenda behind this srs is to describe the system and walk through the functionality and how will it work.


Document Conventions:

For this document, I’ll strictly maintain & follow the IEEE standard. All the color and font will be used and adjusted in user-friendly way so that client can easily grasp and understand the complexities and hierarchy for all functions that have been used.


Intended Audience and Reading Suggestions:

This Software Requirements Specification is designed for any kind of developer who works with related fields and will be very understandable for them.


Product Scope:

For the system, a website, as well as a mobile app, will be developed to catch vast users across the world. I will use cross-platform app development to ensure an easy development process.


References:

SRS IEEE standard format.




Overall Description


Product Perspective:

The perspective of this project is to create a system to take online exams for teachers and students. The project will be created for multi platforms. For android and web.


Product Functions:

The functionality of the product includes:

  • Save user login info
  • Create Question with MCQ and micro-viva
  • Can specify individual marks for specific question
  • Can set specific time for specific questions.
  • Can set automatic markings.
  • Built in speech recognition system will read micro-viva answer
  • Randomizer can be set in question and as well as in answer.


User Classes and Characteristics:

This product will include three user classes:

  • Admin
  • Users
  • Evaluator



Operating Environment:

The operating environments of this product are:

  • Operating System: Windows, Android
  • Hardware Platform: Personal Computer and Mobile Phone
  • Database: Firebase/Django


Design and Implementation Contraints:

Constraints include:

  • The documents should not be handwritten.
  • The supported operating system will be windows and android


User Documentation:

All the actors will be able to see all functions of the project in the dashboard.


Assumptions and Dependencies:

The accracy of the model for the machine learning for speech recognition can be very low sometimes.

Software Interfaces

Communications Interfaces

This softwere will communicate with google login through API for login verification.Will also communicate with sqlite server for data store and update. For notify user through email about any kind of change in database system it will use API connecting google email service with the system.




System Features


Records Management:

  • Save user information
  • Offer different dashboards to different types of actors
  • Allow individual evaluator to manage individual’s exam-cohort
  • The user who starts the cohort will be the evaluator for that specific cohort
  • The evaluator can set multiple questions for the exam.
  • For each question a mark must be specified. Questions can be MCQs or micro-viva.
  • For both the marking should be automatic. For MCQs, the evaluator should mark the correct answer(s).
  • When presenting MCQs to the candidates, the options should be randomized in order.
  • For a micro-viva the question is supplied as a recording and a correct answer is also supplied as a recording by the evaluator.
  • During the exam, the candidate will hear the question because the system will play the recorded question.
  • The system should run a speech recognition solution on the recording and compare the answer with the pre-specified answer for a correct/incorrect assessment.
  • The evaluator can set a maximum time, it is the number of seconds allowed for a student to answer a question before the system moves to the next one.
  • The marking will be done automatically by the system.
  • The evaluator can also adjust the mark of any question of an assessment within the exam cohort.
  • The system will generate a spreadsheet of marks, with full breakdowns of each mark of each assessment for each student.

Accounts Management:

  • There is no limit for signup.
  • Users will be able to reset their passwords by their email.
